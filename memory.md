# 專案歷史錯誤與教訓記憶 (Memory)

本文件紀錄本專案在過去開發過程中所犯的錯誤、踩過的坑，以及後續的解決方案。AI Agent 在開發過程中應主動參考此文件以避免重複犯錯。

## 1. 程式碼邏輯與語法問題
- **`app.js` 代碼汙染與語法錯誤 (v1.6.0)**：過去在修改 `app.js` 時，曾發生嚴重的代碼汙染及 JavaScript 函式閉合 (Closure) 語法錯誤，導致系統崩潰。
  - **調整與解決**：進行了徹底的代碼清理與修復。未來在修改核心邏輯時，需以最小範圍 (Atomic) 變更為主，並嚴格檢查語法閉合。

## 2. 系統穩定度與功能設計
- **不穩定的清除紀錄功能 (v1.6.0)**：曾實作過「選單式清除紀錄」功能，但因狀態管理不穩定而影響了核心對話體驗。
  - **調整與解決**：為確保核心穩定性，已將該功能徹底移除。未來導入新功能前，必須確認其不會破壞既有系統的穩定度。

## 3. 專案流程與分支管理
- **`main` 分支汙染問題 (v1.6.1)**：過去曾因直接在主分支開發，導致 `main` 分支夾雜未驗證的程式碼。
  - **調整與解決**：已於 `Development_Governance.md` 確立 Git 分支與 Worktree 隔離策略，禁止直接在主分支上進行測試與開發。

## 4. 環境與依賴設定問題 (歷史紀錄)
- **命令列工具安裝錯誤**：過去曾在安裝 NVM 及 Claude Code CLI 時，發生 URL 語法錯誤或 PATH 環境變數未正確設定，導致指令無法執行 (`openspec` 執行失敗)。
  - **調整與解決**：已修正安裝流程與 PATH 設定。未來在執行任何環境建置腳本前，需先確認終端機的環境變數及指令語法正確無誤。
