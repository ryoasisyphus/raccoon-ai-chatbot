# Raccoon AI Assistant: 智能客服原型 (Prototype)

![Raccoon Assistant](static/concept.png)

## 🌟 專案簡介
Raccoon AI Assistant 是一個專為一般消費者設計的智能客服聊天機器人原型。
本專案採用 **Vibe Coding** 開發模式，結合現代化 Web 技術與 AI 意圖識別邏輯，提供流暢且具備情感連結的使用者體驗。

### 核心願景
> 「客服不應只是冷冰冰的問答，而是一個能主動引導並解決問題的智能助手。」

---

## 🛠️ 易用性驅動的功能設計 (Usability-Driven Design)

本專案不僅是技術實作，更是從「消費者心理學」出發，解決自動化服務中的常見痛點：

- **智慧意圖引導 (3-Stage Discovery)**
    - **消費者價值**：**「不再有死胡同」**。當使用者輸入模糊時，AI 會主動提供「猜測選項」而非報錯。
    - **為什麼更好**：將開放式問答轉化為引導式點擊，大幅降低使用者的思考負擔與輸入成本。
- **動態真人轉接 (Conditional Handover)**
    - **消費者價值**：**「最後一公里的安全感」**。轉接入口僅在 AI 無法解決問題時動態出現。
    - **為什麼更好**：平衡了「快速自動化」與「人工溫度」，確保複雜問題絕不被遺漏，同時優化客服資源分配。
- **高質感視覺信任 (Visual Trust Design)**
    - **消費者價值**：**「專業且專業的數位體驗」**。採用玻璃擬態與流暢動畫。
    - **為什麼更好**：質感即信任。高品質的介面能有效提升品牌形象，並在推薦商品時增加消費者的轉換信心。
- **零阻力快捷入口 (Contextual Quick Actions)**
    - **消費者價值**：**「所見即所得」**。首頁直接配置高頻需求按鈕（如訂單、禮物）。
    - **為什麼更好**：針對行動端優化，減少打字次數，讓使用者在 3 秒內進入核心服務路徑。

## 📂 專案結構
- `index.html`: 主應用程式檔案（包含 UI 與前端佈局）。
- `src/app.js`: 核心業務邏輯，包含與 Google Sheets API (GAS) 的連線與意圖辨識。
- `static/`: 存放圖片、CSS 樣式表。
- `data/`: 備用本地 Mock 資料 (JSON)，當雲端 API 失敗時作為 Fallback 方案。
- `docs/`:
  - `product_spec.md`: 詳細的產品需求、Prompt 設計、邏輯架構與未來路徑。
  - `walkthrough.md`: 專案實作紀錄與操作指南。
- `prompts/`:
  - `01_product_scoping.md`: 用於指導 AI 開發此專案的優化提示詞。

## 🚀 如何啟動
1. 複製此專案至本地環境。
2. 直接使用瀏覽器開啟根目錄下的 `index.html`。
3. **無須任何安裝步驟 (Zero-Config)**。

---

## 🧭 測試路徑建議
1. **探索階段**：使用底部的快捷按鈕，測試「運費資訊」或「訂單查詢」。
2. **購買階段**：輸入「我想買點東西」或「推薦禮物」，查看商品卡片展示。
3. **挫折階段 (測試引導邏輯)**：
   - 輸入「你是誰？」（觸發猜測問題）。
   - 再次輸入無關內容（觸發真人轉接按鈕）。

## 🔮 未來展望
- **擴展雲端功能 (Cloud Scaling)**：基於目前的 Google Sheets Serverless 架構，後續可進一步擴增不同工作表來管理活動檔期或更多客製化回覆。
- **接入真 LLM (Gemini/GPT)**：搭配現有的意圖辨識，實現真正的自然語言長文理解與更複雜的推理。
- **多平台支援**：將目前的前端 UI 與後端邏輯部署整合至 LINE / WhatsApp / FB Messenger。
